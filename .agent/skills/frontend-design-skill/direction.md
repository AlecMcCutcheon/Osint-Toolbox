# Direction Synthesis

After discovery and research, translate findings into clear design directions.

Do not jump from research straight to implementation. First define the viable paths and explain why one should win.

## Candidate directions

When the brief is non-trivial, propose 2-3 candidate directions.

Each direction should include:
- short name
- one-sentence concept
- who it serves best
- why it fits the product and context
- visual language
- UX characteristics
- risks or tradeoffs

## What a direction must specify

### Visual language
- typography approach
- spacing and density
- layout system
- color logic
- imagery or illustration style
- motion philosophy
- component personality

### UX intent
- what gets emphasis first
- how the main task is made obvious
- how cognitive load is reduced
- how trust or clarity is established
- how the design handles dense content or low-attention use

## Recommendation rule

Do not present all directions as equal if one is clearly better.

Recommend one direction and explain:
- why it best fits the user and task
- what tradeoffs it makes
- what assumptions it depends on
- what could change the recommendation

## Example decision framing

- Direction A is best when conversion and clarity matter most.
- Direction B is best when brand differentiation and editorial storytelling matter most.
- Direction C is best when power-user density and fast scanning matter most.

## Assumption labeling

Separate:
- confirmed facts
- inferred conclusions
- open assumptions

Never bury assumptions inside confident design language.

## Pre-build structure

Before writing code, define:
- primary screen or page hierarchy
- main sections or modules
- primary CTA and secondary actions
- state coverage: loading, empty, error, success, disabled
- responsive shifts from mobile to desktop
- what makes the result distinctive beyond cosmetics

## Direction quality bar

A valid direction should feel like it was designed for this exact product, audience, and task.

If you can swap the logo and reuse the exact same direction for ten unrelated products, it is too generic.
