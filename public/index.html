Jobs run one at a time in list order. The queue splits into Lines (numbers) and Names (people-search result sets), plus People (profile enrich jobs), each in its own box. The app cache keeps repeat numbers fast (no new Flare run when cached). Each Enqueue adds a row; the Cache tag means that run was served from the app cache, not a separate data source. The queue stays in the browser until you clear site data. The server graph is rebuilt to match that list; an empty list clears the graph.
